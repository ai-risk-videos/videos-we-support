# Videos We Support

A tool for helping great creators make great videos about AI risk.

You paste a YouTube channel, and the tool studies how that creator actually talks (from their recent transcripts) and generates a tailored list of AI-risk video ideas in their voice. You trim and tweak the list, then publish a private link to send the creator. For any idea, the creator can open a documented research pack (real, cited sources and cases) and a sample script written in their style.

**Live app:** https://videos-we-support.web.app
**How it works (auto-updated from the running code):** https://videos-similar-api-production.up.railway.app/pipeline

## Why this exists

Public understanding of AI risk lags far behind where the technology is. The highest-leverage way to close that gap is not another report, it is helping the creators people already trust make accurate, compelling videos. This tool lowers the effort for a creator to say yes: they get ideas that fit their channel, sources they can verify, and a draft they can rewrite in their own words.

The content is grounded in documented, real cases and the words of the researchers who built these systems. It is awareness and education, never operational instructions.

## How it works

1. **Understand the creator.** Fetch the channel, pull recent captioned transcripts, and build a short profile of the creator's voice, format, and audience.
2. **Generate ideas.** Write fresh ideas tailored to that channel, or rank the existing library of documented AI-risk leads against the channel.
3. **Flesh out an idea.** On demand, build a research pack (a cited brief drawn from a curated source bank and an incident library) and a sample script in the creator's voice.
4. **Curate and publish.** Trim the list, add a note, and publish a private page. The link is what you send the creator. Research packs and sample scripts can be pre-made so the creator sees them instantly.

The `/pipeline` page above documents every step, prompt, and data source, and it is generated from the running backend so it never goes stale.

## Running it yourself

**Frontend** (static, no build step beyond the generator):

```bash
cd frontend
python3 build_leads.py            # writes species-web-deploy/index.html
# serve species-web-deploy/ with any static server, or deploy to Firebase Hosting
```

**Backend** (FastAPI):

```bash
cd backend
pip install -r requirements.txt
cp ../.env.example .env            # then fill in your keys (see below)
uvicorn main:app --reload
```

The backend needs an `ANTHROPIC_API_KEY`. Transcript fetching for arbitrary channels also needs Webshare proxy credentials. See [.env.example](.env.example) for the full list. No secrets are committed to this repo; everything is read from environment variables.

## Tech stack

- **Frontend:** a single self-contained HTML page generated by a Python script, hosted on Firebase Hosting.
- **Backend:** FastAPI on Railway, calling Claude for generation.
- **Storage:** Firestore holds the published creator pages and their pre-made packs and scripts.
